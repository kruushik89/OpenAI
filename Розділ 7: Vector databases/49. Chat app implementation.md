# Chat app: реалізація «chat with your data»

У цій лекції — **готовий застосунок** «chat with your own data»: зберігаємо дані з embeddings у ChromaDB, шукаємо релевантний chunk і відповідаємо через OpenAI з **context injection**.

**Вимога:** ChromaDB має бути **запущена** (у курсі — через Docker).

---

## Тестові дані

Три записи про вигаданий світ (ручні ID):

| ID / запис | Зміст |
|------------|--------|
| **Student Info** | Базова інформація про **Alexandra Thompson** (вільний час, хобі тощо) |
| **Club Info** | Інформація про клуб |
| **University Info** | Інформація про університет |

Це **starting data** — невеликі chunks, які застосунок індексує в vector DB.

---

## Структура коду

1. **Embedding function** — та сама OpenAI embedding function з попередніх лекцій.
2. **Функції для Chroma API:**
   - `createCollection` — створити collection;
   - `getCollection` — отримати collection;
   - `populateCollection` — додати documents (`student info`, `club info`, `university info`) разом з **ids** (створені вручну).
3. **`askQuestion`** — головний flow: запит → similarity search → OpenAI.

Файл у проєкті: `embeddings/src/chroma/RelevantInfo.ts`.

Запуск (з папки `embeddings`, ChromaDB у Docker має бути активна):

```bash
npm start
```

Скрипт у `package.json`:

```json
"start": "node -r ts-node/register --env-file=.env src/chroma/RelevantInfo.ts"
```

Потрібен `.env` з `OPENAI_API_KEY`.

## Повний код з поясненнями

### Imports і клієнт

```ts
import { ChromaClient } from "chromadb";
import { OpenAIEmbeddingFunction } from '@chroma-core/openai';
import OpenAI from "openai";

const chroma = new ChromaClient({
    host: 'localhost',
    port: 8000,
});
```

- **`ChromaClient`** — підключення до сервера ChromaDB (Docker на порту `8000`).
- **`OpenAIEmbeddingFunction`** — embeddings через OpenAI (окремий пакет `@chroma-core/openai` у chromadb v3).
- **`OpenAI`** — Chat Completions для фінальної відповіді (context injection).
- У v3 використовуємо **`host` + `port`**, а не застарілий `path: "http://localhost:8000"`.

### Тестові дані (3 chunks)

```ts
const studentInfo = `Alexandra Thompson, a 19-year-old computer science sophomore with a 3.7 GPA,
is a member of the programming and chess clubs who enjoys pizza, swimming, and hiking
in her free time in hopes of working at a tech company after graduating from the University of Washington.`;

const clubInfo = `The university chess club provides an outlet for students to come together and enjoy playing
the classic strategy game of chess. Members of all skill levels are welcome, from beginners learning
the rules to experienced tournament players. The club typically meets a few times per week to play casual games,
participate in tournaments, analyze famous chess matches, and improve members' skills.`;

const universityInfo = `The University of Washington, founded in 1861 in Seattle, is a public research university
with over 45,000 students across three campuses in Seattle, Tacoma, and Bothell.
As the flagship institution of the six public universities in Washington state,
UW encompasses over 500 buildings and 20 million square feet of space,
including one of the largest library systems in the world.`;
```

Три незалежні текстові фрагменти — майбутні **documents** у vector DB. Кожен chunk індексується окремо; при запиті Chroma поверне найближчий за семантикою.

### Embedding function і ім'я collection

```ts
const embeddingFunction: OpenAIEmbeddingFunction = new OpenAIEmbeddingFunction({
    apiKey: process.env.OPENAI_API_KEY!,
    modelName: 'text-embedding-3-small'
})

const collectionName = "personal-infos";
```

При `add` і `query` Chroma **сама** викличе OpenAI Embeddings API — передавати `embeddings` вручну не потрібно.

### `createCollection` — створення collection

```ts
async function createCollection() {
    try {
        await chroma.deleteCollection({ name: collectionName });
    } catch {
        // collection doesn't exist yet
    }

    await chroma.createCollection({
        name: collectionName,
        embeddingFunction,
    });
}
```

**Важливо:** при `createCollection` обов'язково передати **`embeddingFunction`**. Якщо collection створити без неї, на сервері збережеться `embedding_function: { name: "default" }`, і клієнт почне шукати пакет `@chroma-core/default-embed` (попередження в консолі).

`deleteCollection` перед створенням — щоб перестворити collection з правильним конфігом під час навчального демо. У продакшені зазвичай не видаляють дані на кожному запуску.

### `getCollection` і `populateCollection` — запис у БД

```ts
async function getCollection() {
    const collection = await chroma.getCollection({
        name: collectionName,
        embeddingFunction
    });
    return collection;
}

async function populateCollection() {
    const collection = await getCollection();
    await collection.add({
        documents: [studentInfo, clubInfo, universityInfo],
        ids: ['id1', 'id2', 'id3'],
    })
}
```

- **`getCollection`** — отримує collection з тим самим `embeddingFunction`, що й при створенні.
- **`populateCollection`** — додає 3 documents з ручними **ids**. Embeddings генеруються автоматично через OpenAI.

### `askQuestion` — retrieval + context injection

```ts
async function askQuestion() {
    // const question = 'What does Alexandra Thompson like to do in her free time?';
    // const question = 'What is the University of Washington?';
    const question = 'How old is Alexandra Thompson?';
    const collection = await getCollection();
    const result = await collection.query({
        queryTexts: [question],
        nResults: 1
    })
    console.log('result', result);
    const relevantInfo = result.documents[0][0]
    if (relevantInfo) {
        const openai = new OpenAI();
        const response = await openai.chat.completions.create({
            model: 'gpt-3.5-turbo',
            temperature: 0,
            messages: [{
                role: 'assistant',
                content: `Answer the next question using this information: ${relevantInfo}` // context injection
            },
            {
                role:'user',
                content: question
            }]
        })
        const responseMessage = response.choices[0].message;
        console.log(responseMessage.content);
    }
}
```

Кроки всередині функції:

1. **`collection.query`** — similarity search: питання → embedding → найближчі documents.
2. **`result.documents[0][0]`** — перший (найрелевантніший) chunk для першого запиту.
3. **Context injection** — знайдений текст підставляється в prompt перед питанням.
4. **`temperature: 0`** — детерміновані відповіді для демо.

Можна міняти `question` — Chroma підбере інший chunk (student / club / university info).

### `main` — два режими запуску

```ts
async function main() {

    // await createCollection();
    // await populateCollection();
    await askQuestion();

}

main();
```

| Фаза | Що розкоментувати в `main()` | Результат |
|------|-------------------------------|-----------|
| **1. Заповнення БД** | `createCollection()` + `populateCollection()` | Collection `personal-infos` з 3 documents |
| **2. Запитання** | лише `askQuestion()` | Відповідь GPT на основі знайденого chunk |

Спочатку запускаємо **фазу 1**, потім коментуємо її і запускаємо **фазу 2**.

## Flow: від питання до відповіді

```
1. createCollection() + populateCollection()
2. Питання користувача
3. collection.query({ queryTexts: [...] })
4. relevantInfo = results.documents
5. OpenAI Chat Completions + контекст + те саме питання
6. Вивід відповіді
```

### 1. Заповнення БД

```ts
await createCollection();
await populateCollection();
// npm start — collection створена і заповнена
```

### 2. Запит до collection

Питання передається як **query text** — рядок або **масив рядків** (можна одразу кілька запитів і отримати найкращі результати для кожного).

**Приклад питання:** *What does Alexandra Thompson like to do in her free time?*

Відповідь міститься в одному з entries (**Student Info**).

### 3. Релевантний chunk

```ts
const relevantInfo = results.documents;
```

`documents` — найближчі записи з vector DB до embedding питання. У debug видно, що повертається **саме той chunk**, який містить відповідь (наприклад, про swimming, hiking, pizza).

### 4. Context injection → OpenAI

Якщо `relevantInfo` не порожній:

- Prompt на кшталт: **«Answer the next question using this information»** + **контекст** (знайдений chunk) + **те саме питання**.
- Це і є **context injection** з попередньої лекції.
- Друкуємо відповідь моделі.

**Результат демо:** модель відповідає, що Alexandra Thompson у вільний час любить *swimming, hiking and eating pizza* — хоча в сирому тексті chunk може бути просто опис без явної формули «це про Alexandra Thompson».

## Що повертає Chroma при query

У debug на breakpoint `relevantInfo`:

| Поле | Опис |
|------|------|
| **`documents`** | Тексти entries, найближчі до запиту |
| **`distances`** | «Відстань» / score схожості — як у попередньому розділі (similarity score) |

Параметр **`nResults`** — скільки результатів повернути. У демо беруть **перший** (найрелевантніший); можна запросити **кілька** — для кожного буде свій `distance`.

| Крок | Дія | Результат |
|------|-----|-----------|
| 1 | `createCollection` + `populateCollection`, `npm start` | Collection створена, 3 documents з embeddings |
| 2 | `askQuestion`, `npm start` | Коректна відповідь про хобі Alexandra Thompson |

---

## Підсумок pipeline

```
Student / Club / University info
        ↓ populate (documents + ids + embeddings)
    ChromaDB collection
        ↓ query(queryTexts: question)
    documents + distances
        ↓ context injection
    OpenAI Chat Completions
        ↓
    Природна відповідь на питання
```

Це повна реалізація **RAG-подібного** flow на ChromaDB: retrieval (similarity search) + generation (OpenAI з контекстом).
