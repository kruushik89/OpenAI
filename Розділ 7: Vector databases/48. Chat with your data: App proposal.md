# Chat with your data: пропозиція застосунку

У цій лекції — **пропозиція застосунку** на базі embeddings: як працює додаток типу «chat with your data» і який pipeline ми реалізуємо далі.

---

## Навіщо vector DB для запитів

До цього ми зберігали embeddings у vector database. Тепер їх можна **шукати за схожістю** — не текстовою (string similarity), а **векторною**: dot product, cosine similarity тощо.

Це те саме, що ми робили вручну в попередньому розділі, але **на рівні бази даних** і в масштабі.

---

## Як працює запит до vector DB

```
Питання (query string)
        ↓
  Embedding model → embedding запиту
        ↓
  Similarity search (dot / cosine) у vector DB
        ↓
  Найближчі (найрелевантніші) результати
```

1. Беремо **рядок-запит** (питання користувача).
2. Через **embedding model** будуємо embedding цього запиту.
3. У vector DB виконуємо **similarity search** — знаходимо найближчі записи.
4. База повертає **найрелевантніші** chunks / документи.

---

## Context injection (ін’єкція контексту)

Ключова ідея застосунку — **context injection**: дати моделі OpenAI **контекст із ваших приватних даних**, яких не було в навчанні.

**Приклад:** стара книга з домашньої бібліотеки, PDF з інформацією про людину, якої не було у training data OpenAI.

**Pipeline:**

1. Розбити великий документ на **невеликі chunks**.
2. Зберегти chunks **разом з embeddings** у vector database.
3. Користувач **ставить питання** — воно стає query для БД.
4. **Similarity search** знаходить найрелевантніший chunk.
5. Цей chunk передаємо в **OpenAI** як контекст — модель відповідає на питання, спираючись на ваші дані.

Так можна отримувати відповіді з **приватних / локальних** джерел, не покладаючись лише на знання моделі з навчання.

**Повний flow застосунку:**

```
Приватні дані → chunks → embeddings → ChromaDB
                                              ↓
Питання → embedding запиту → similarity search → найрелевантніший chunk
                                              ↓
                              OpenAI + контекст → відповідь
```

Майже весь код і дані вже будуть готові; залишиться дописати **запити до ChromaDB**. Застосунок буде **простим** — детальна реалізація в наступній лекції.
