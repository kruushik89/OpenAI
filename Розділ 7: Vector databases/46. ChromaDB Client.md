# ChromaDB Client: перші операції

ChromaDB server уже запущений у Docker. У цій лекції створюємо **клієнт** для операцій з БД.

## Підготовка

1. Встановити npm-пакет (уважність до typo в назві):

```bash
npm install chromadb
```

Пакет підтримує **TypeScript**.

2. Створити папку `chroma` і файл `client.ts`.

## Підключення клієнта

```ts
import { ChromaClient } from 'chromadb';

const client = new ChromaClient({
  host: 'localhost',
  port: 8000,
});
```

> У chromadb v3 аргумент `path` застарілий — використовуй `host` / `port` / `ssl`.

Операції **асинхронні**. Якщо немає top-level await — обгортаємо логіку в `async function main()`.

## Створення collection

```ts
const response = await client.getOrCreateCollection({
  name: 'data-test',
  // embeddings передаємо вручну — без DefaultEmbeddingFunction
  embeddingFunction: null,
});
```

- `createCollection` — лише якщо collection ще немає (інакше `ChromaUniqueError: The resource already exists`).
- `getOrCreateCollection` — безпечніше для повторних запусків.
- Без `embeddingFunction: null` клієнт спробує `@chroma-core/default-embed` (для цієї лекції не потрібен).

У відповіді — ім’я collection і **ID** (потрібен для подальших запитів).

### Перевірка в браузері

- Collections: `http://localhost:8000/api/v1/collections` (у курсі; у нових версіях API може бути **v2**)
- Документація / Swagger усіх REST-операцій: `http://localhost:8000/docs`

## Додавання даних

```ts
async function addData() {
  const collection = await client.getCollection({ name: 'data-test' });

  await collection.add({
    ids: ['id0'],
    documents: ['Here is my entry'],
    embeddings: [[/* ручний тестовий вектор */]],
  });
}
```

Параметри `add`:

| Параметр | Опис |
|----------|------|
| `ids` | масив ID записів |
| `documents` | масив текстів |
| `embeddings` | вектори (у цій лекції — **вручну**, для тесту) |

## Два способи роботи з embeddings

1. **Передавати embeddings вручну** при `add` — як у цій лекції.
2. **Не передавати** — тоді при створенні collection треба задати **embedding function**. Інакше помилка: `embedding function is undefined`.

У наступній лекції — embedding function через **OpenAI**, щоб embeddings генерувались автоматично при додаванні даних.
