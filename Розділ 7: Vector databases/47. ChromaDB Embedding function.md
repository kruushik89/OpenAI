# ChromaDB: embedding function з OpenAI

У попередній лекції embeddings передавали **вручну** при `add`. Тут — **автоматична генерація** через OpenAI всередині ChromaDB.

## Контекст з документації Chroma

Chroma розділяє пакети для **Python** і **JavaScript**. JS-версія наразі **обмеженіша**: наприклад, **Hugging Face embeddings** недоступні. У цій лекції — **OpenAI embeddings** напряму з Chroma.

Процес простий: при створенні collection задаємо **embedding function**, і при `add` достатньо передати лише **documents** — вектори Chroma згенерує сама через OpenAI API.

---

## Нова collection (важливо)

Collection, створену **без** embedding function (як `data-test` з `embeddingFunction: null`), **не можна** просто «дописати» embedding function пізніше — навіть якщо в документації здається, що можна, на практиці це не працює (є баги).

**Рішення:** створити **нову** collection з **новим ім’ям**, наприклад `data-test-2`.

```ts
async function main() {
  await client.createCollection({
    name: 'data-test-2',
    embeddingFunction,
  });
}
```

Спочатку запускаємо `main()` (`npm start`), щоб collection з’явилась на сервері.

---

## OpenAI embedding function

Додатковий import (у курсі — з пакета Chroma):

```ts
import { OpenAIEmbeddingFunction } from 'chromadb';
```

> У **chromadb v3** OpenAI embedding function може бути в окремому пакеті `@chroma-core/openai` — перевір актуальну документацію Chroma для JS.

Створення функції:

```ts
const embeddingFunction = new OpenAIEmbeddingFunction({
  openai_api_key: process.env.OPENAI_API_KEY!,
  openai_model: 'text-embedding-3-small', // опційно
});
```

| Параметр | Опис |
|----------|------|
| `openai_api_key` | ключ з `.env` (`OPENAI_API_KEY`) |
| `openai_model` | модель embeddings; у лекції — **text-embedding-3-small** (на момент запису — нова; згодом може стати дефолтною) |

### TypeScript і `!`

`process.env.OPENAI_API_KEY` має тип `string | undefined`. TypeScript попереджає, що значення може бути `undefined`. Оскільки ключ **обов’язковий**, автор додає **`!`** (non-null assertion): «ми знаємо, що змінна задана в `.env`».

## Створення collection з embedding function

При `createCollection` / `getOrCreateCollection` **обов’язково** передати `embeddingFunction`:

```ts
await client.createCollection({
  name: 'data-test-2',
  embeddingFunction,
});
```

Без цього параметра collection не зможе сама генерувати embeddings (автор у відео забув указати його з першого разу — другий запуск уже спрацював).

---

## Додавання даних без ручних embeddings

```ts
async function addData() {
  const collection = await client.getCollection({
    name: 'data-test-2',
    embeddingFunction, // той самий instance, що при створенні
  });

  await collection.add({
    ids: ['id0'],
    documents: ['Here is my entry'],
    // embeddings не передаємо — Chroma викличе OpenAI API
  });
}
```

Після `npm start` під капотом ChromaDB викликає **OpenAI Embeddings API** для рядка `Here is my entry` і зберігає отриманий вектор у collection.

`collection.add()` повертає `void` — успіх = відсутність помилки, не JSON у консолі.

## Перевірка вмісту БД

Подивитись «що всередині» через браузер **не так просто**, як heartbeat або список collections. Зручніше збирати невеликий застосунок — наприклад, **перевірка similarity** (пошук схожих записів).
