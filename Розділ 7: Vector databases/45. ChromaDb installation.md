# Встановлення ChromaDB через Docker

Найпростіший спосіб підняти **ChromaDB server** локально — через **Docker**. Для цієї лекції потрібен встановлений Docker.

## Кроки

### 1. Pull образу

З документації ChromaDB: спочатку **pull** образу з Docker Hub, потім запустити контейнер.

```bash
docker pull chromadb/chroma
```

### 2. Запуск контейнера

```bash
docker run -p 8000:8000 chromadb/chroma
```

Важливо: **пробросити порт 8000** (`-p 8000:8000`), щоб звертатися до сервера з хоста.

У контейнері стартує **Python-процес** ChromaDB.

## Перевірка

- У Docker-клієнті / **VS Code Docker extension** видно запущений контейнер (автор надає перевагу саме цьому розширенню).
- У браузері відкрити: `http://localhost:8000/api/v2/heartbeat`
- Якщо все ок — з’явиться JSON на кшталт `{"nanosecond heartbeat": ...}` → сервер успішно піднято.
- У логах контейнера також видно активність.

> **Примітка (актуальний образ):** у курсі перевіряли `http://localhost:8000/api/v1`, але в новішому Chroma **v1 API deprecated**. Корінь `http://localhost:8000/` теж дає 404 — це нормально, окремої веб-сторінки немає.

Після цього можна писати **Chroma client** застосунок.
