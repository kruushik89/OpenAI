# Tool parameters — параметри функцій (tools)

У цій лекції переходимо на **наступний рівень**: викликаємо функції не лише без аргументів, а й **з параметрами**, які модель витягує з промпту користувача.

Приклад сценарію: у бекенді є база даних зі статусами замовлень, і ми хочемо **запитувати її через OpenAI** — це відкриває набагато більше можливостей.

---

## 1. Функція `getOrderStatus`

Створюємо «заглушку», яка імітує доступ до захищеної бази даних:

```ts
function getOrderStatus(orderId: string): string {
  console.log(`Getting order status for order ${orderId}`);
  const orderAsNumber = parseInt(orderId);

  if (orderAsNumber % 2 === 0) {
    return "IN_PROGRESS";
  }

  return "COMPLETED";
}
```

- Параметр `orderId` — рядок, перетворюємо на число через `parseInt`.
- Умовна логіка для демо: **парне** число → `"IN_PROGRESS"`, **непарне** → `"COMPLETED"`.

## 2. Оновлення system prompt

Модель має знати, що вона працює не лише з часом доби, а й зі статусами замовлень:

```ts
{
  role: "system",
  content:
    "You are a helpful assistant that gives information about the time of the day and the status of an order.",
}
```

## 3. Новий user prompt

```ts
{
  role: "user",
  content: "What is the time of the day? And what is the status of the order 123?",
}
```

Модель має зрозуміти, що `123` — це `orderId`, і передати його у функцію `getOrderStatus`.

## 4. Tool з параметрами — головна новинка

Порівняно з попередньою лекцією з'являється нове поле — **`parameters`**.

```ts
{
  type: "function",
  function: {
    name: "getOrderStatus",   // має точно збігатися з ім'ям функції в коді
    description: "Get the status of an order",
    parameters: {
      type: "object",         // наразі працює лише "object"
      properties: {
        orderId: {
          type: "string",
          description: "The ID of the order to get the status of",
        },
      },
      required: ["orderId"],  // обов'язкові параметри
    },
  },
}
```

### Важливі деталі

| Поле | Навіщо |
|------|--------|
| `name` | Ідентифікатор функції — **без помилок у написанні** |
| `description` | Допомагає моделі зрозуміти, коли викликати tool |
| `parameters.type` | Зазвичай лише `"object"` — `number` або `string` на верхньому рівні не працюють |
| `properties` | Опис кожного параметра: ім'я, тип, опис |
| `required` | Масив обов'язкових параметрів; тут — `["orderId"]` |

> TypeScript поки не дає автодоповнення для `properties` — описуємо схему вручну.

## 5. Витягування аргументів з відповіді API

Коли функція приймає параметри, у `toolCall` з'являється поле **`arguments`** — рядок у форматі JSON. Його потрібно розпарсити:

```ts
} else if (toolName === "getOrderStatus") {
  const parsedArguments = JSON.parse(toolCall.function.arguments);
  toolResponse = getOrderStatus(parsedArguments.orderId);
}
```

Порядок дій:

1. Отримати `toolCall.function.arguments` (сирий рядок).
2. Розпарсити через `JSON.parse`.
3. Витягнути потрібні поля (`orderId`).
4. Передати їх у локальну функцію.

## 6. Два tools і обробка кількох tool calls

У `tools` тепер **два** записи — `getTimeOfDay` і `getOrderStatus`. Якщо промпт запитує і час, і статус замовлення, модель може повернути **кілька** `tool_calls` одночасно. Тому обробляємо **усі** виклики в циклі:

```ts
const willInvokeFunction = response.choices[0].finish_reason === "tool_calls";
const toolCalls = response.choices[0].message.tool_calls ?? [];

if (willInvokeFunction) {
  context.push(response.choices[0].message);

  for (const toolCall of toolCalls) {
    if (toolCall.type !== "function") continue;

    const toolName = toolCall.function.name;
    let toolResponse: string;

    if (toolName === "getTimeOfDay") {
      toolResponse = getTimeOfDay();
    } else if (toolName === "getOrderStatus") {
      const parsedArguments = JSON.parse(toolCall.function.arguments);
      toolResponse = getOrderStatus(parsedArguments.orderId);
    } else {
      continue;
    }

    context.push({
      role: "tool",
      content: toolResponse,
      tool_call_id: toolCall.id,
    });
  }
}
```

> **Важливо:** повідомлення асистента з `tool_calls` додаємо **один раз**, а для **кожного** `tool_call_id` — окрему відповідь `role: "tool"`. Інакше другий запит до API поверне помилку 400.

Далі — як у попередній лекції: робимо **другий виклик** OpenAI з оновленим контекстом.

## 7. Повний код `tools/index.ts`

```ts
import OpenAI from "openai";

const openAI = new OpenAI();

function getTimeOfDay(): string {
  return new Date().toLocaleTimeString();
}

function getOrderStatus(orderId: string): string {
  console.log(`Getting order status for order ${orderId}`);
  const orderAsNumber = parseInt(orderId);

  if (orderAsNumber % 2 === 0) {
    return "IN_PROGRESS";
  }
  return "COMPLETED";
}

async function callOpenAIWithTools() {
  const context: OpenAI.Chat.ChatCompletionMessageParam[] = [
    {
      role: "system",
      content:
        "You are a helpful assistant that gives information about the time of the day and the status of an order.",
    },
    {
      role: "user",
      content: "What is the time of the day? And what is the status of the order 123?",
    },
  ];
  const response = await openAI.chat.completions.create({
    model: "gpt-4o-mini",
    messages: context,
    tools: [
      {
        type: "function",
        function: {
          name: "getTimeOfDay",
          description: "Get the current time of the day",
        },
      },
      {
        type: "function",
        function: {
          name: "getOrderStatus",
          description: "Get the status of an order",
          parameters: {
            type: "object",
            properties: {
              orderId: {
                type: "string",
                description: "The ID of the order to get the status of",
              },
            },
            required: ["orderId"],
          },
        },
      },
    ],
    tool_choice: "auto",
  });

  const willInvokeFunction = response.choices[0].finish_reason === "tool_calls";
  const toolCalls = response.choices[0].message.tool_calls ?? [];

  if (willInvokeFunction) {
    context.push(response.choices[0].message);

    for (const toolCall of toolCalls) {
      if (toolCall.type !== "function") continue;

      const toolName = toolCall.function.name;
      let toolResponse: string;

      if (toolName === "getTimeOfDay") {
        toolResponse = getTimeOfDay();
      } else if (toolName === "getOrderStatus") {
        const parsedArguments = JSON.parse(toolCall.function.arguments);
        toolResponse = getOrderStatus(parsedArguments.orderId);
      } else {
        continue;
      }

      context.push({
        role: "tool",
        content: toolResponse,
        tool_call_id: toolCall.id,
      });
    }
  }

  const secondResponse = await openAI.chat.completions.create({
    model: "gpt-4o-mini",
    messages: context,
  });

  console.log(secondResponse.choices[0].message.content);
}

callOpenAIWithTools();
```

## 8. Результат

Після `npm start`:

| Запит | Результат |
|-------|-----------|
| *What is the status of the order 123?* | `123` — непарне → статус **COMPLETED** |
| *What is the status of the order 1234?* | `1234` — парне → статус **IN_PROGRESS** |
| *What is the time of the day? And what is the status of the order 123?* | викликаються **обидва** tools, модель відповідає і про час, і про статус |

Модель сама витягує `orderId` з промпту, викликає `getOrderStatus` з правильним аргументом, а потім формує зрозумілу відповідь користувачу.
