## 2. Теоретична основа (лекція 35)

### Embedding space
Простір, у якому **схожі об'єкти розташовані ближче одне до одного**. Embedding конкретного об'єкта — це просто координати точки в цьому просторі (вектор чисел).

**Приклад із тваринами:** на 2D-графіку з осями *friendliness* та *size* кіт і собака розташовані близько (дружелюбні, невеликі), а ведмідь, вовк, кобра — далі (менш дружелюбні). Точки, що стоять близько, утворюють **кластери**.

### Два способи вимірювання схожості

1. **Dot product (скалярний добуток)** — сума попарних добутків компонентів двох векторів. Фактично показує «відстань» між точками, але залежить від обраних вимірів (dimensions).
2. **Cosine similarity (косинусна схожість)** — вимірює **кут** між векторами (лініями від початку координат до точки). Малий кут → висока схожість (кіт–собака), великий кут → низька (кіт–вовк), протилежні напрямки → майже протилежні вектори (кіт–муха).

### Моделі embeddings від OpenAI

| Модель | Виміри | Особливості |
|--------|--------|-------------|
| `text-embedding-ada-002` | — | стара, популярна, але дорожча |
| `text-embedding-3-small` | 1536 (до ~3000 підтримується серією) | дешевша й продуктивніша за ada-002; **обрана для курсу**; 8192 max tokens |
| `text-embedding-3-large` | — | більш потужна, дорожча |

Порівняння моделей різних провайдерів — на **Hugging Face MTEB leaderboard**.

### Навіщо потрібні embeddings
1. **Similarity search** — пошук за схожістю.
2. **Recommendation systems** — рекомендаційні системи.
3. **Basic chat functionality** — базова функціональність чату (основа для RAG).

## 3. Практична реалізація

Створено окремий TypeScript-проєкт `embeddings/` (ts-node + бібліотека `openai`).

### 3.1. Генерація embeddings (лекція 36)

```typescript
const response = await openai.embeddings.create({
  input,   // string | string[]
  model: "text-embedding-3-small",
});
```

- Якщо `input` — рядок → у `response.data` один embedding.
- Якщо `input` — масив рядків → по одному embedding на кожен елемент масиву.
- Кожен embedding — масив із ~1536 чисел у діапазоні приблизно від -1 до 1.

### 3.2. Збереження embeddings на диск (лекція 37)

Оскільки в розділі не використовується vector database, embeddings зберігаються самостійно через Node.js `fs`:

- `loadJSONData<T>()` — читає та парсить JSON-файл (generic для type safety).
- `saveDataToJSONFile()` — записує результат у JSON.
- Вхідні дані (`data.json`) — список назв тварин/рослин/комах.
- Результат (`dataWithEmbeddings.json`) — масив `{ input, embedding }`.

Фінальний код (файл `main.ts`, у проєкті — з невеликим рефакторингом порівняно з конспектом лекції):

```1:51:/Users/oleksandrkrushynskyi/education/OpenAI/embeddings/src/main.ts
import OpenAI from "openai";
import { readFileSync, writeFileSync } from "fs";
import { join } from "path";

const openai = new OpenAI()

export type DataWithEmbeddings = {
    input: string,
    embedding: number[]
}

export async function generateEmbeddings(input: string | string[]) {
    const response = await openai.embeddings.create({
        model: "text-embedding-3-small",
        input,
    });
    return response.data;
}

export function loadJSONData<T>(filePath: string): T {
    const path = join(__dirname, filePath);
    const rowData = readFileSync(path);
    return JSON.parse(rowData.toString());
}

function saveDataToJSONFile(data: any, fileName: string) {
    const dataString = JSON.stringify(data);
    const dataBuffer = Buffer.from(dataString);
    const path = join(__dirname, fileName);
    writeFileSync(path, dataBuffer);
    console.log(`Data saved to ${fileName}`);
}

export async function saveEmbeddingsFromDataFile(sourceFile: string, outputFile: string) {
    const data = loadJSONData<string[]>(sourceFile);
    const embeddings = await generateEmbeddings(data);
    const dataWithEmbeddings: DataWithEmbeddings[] = data.map((input, i) => ({
        input,
        embedding: embeddings[i].embedding,
    }));
    saveDataToJSONFile(dataWithEmbeddings, outputFile);
}

async function main() {
    await saveEmbeddingsFromDataFile("data.json", "dataWithEmbeddings.json");
    await saveEmbeddingsFromDataFile("data2.json", "dataWithEmbeddings2.json");
}

if (require.main === module) {
    main();
}
```

### 3.3. Розрахунок similarity (лекція 38)

У файлі `similar.ts` реалізовано обидві метрики:

```1:44:/Users/oleksandrkrushynskyi/education/OpenAI/embeddings/src/similar.ts
import { DataWithEmbeddings, generateEmbeddings, loadJSONData } from "./main";

export function dotProduct(a: number[], b: number[]) {
    return a.map((value, index) => value * b[index]).reduce((a, b) => a + b, 0);
}

function cosineSimilarity(a: number[], b: number[]) {
    const product = dotProduct(a, b);
    const aMagnitude = Math.sqrt(a.map(value => value * value).reduce((a, b) => a + b, 0));
    const bMagnitude = Math.sqrt(b.map(value => value * value).reduce((a, b) => a + b, 0));
    return product / (aMagnitude * bMagnitude);
}

async function main() {
    const dataWithEmbeddings = loadJSONData<DataWithEmbeddings[]>('dataWithEmbeddings2.json')

    const input = 'rabbit';

    const inputEmbedding = await generateEmbeddings(input);

    const similarities: {
        input: string,
        similarity: number
    }[] = [];

    for (const entry of dataWithEmbeddings) {
        const similarity = dotProduct(
            entry.embedding,
            inputEmbedding[0].embedding
        )
        similarities.push({
            input: entry.input,
            similarity
        })
    }

    console.log(`Top 10 most similar entries:`)
    const sortedSimilarities = similarities.sort((a, b) => b.similarity - a.similarity);
    sortedSimilarities.forEach(similarity => {
        console.log(`${similarity.input}: ${similarity.similarity}`);
    })
}

main();
```

**Алгоритм пошуку (semantic search):**
1. Завантажити збережені embeddings з JSON.
2. Згенерувати embedding для текстового запиту.
3. Порахувати similarity (dot product або cosine) між запитом і кожним записом.
4. Відсортувати за спаданням — найбільш схожі записи зверху.

### 3.4. Аналіз результатів (лекція 39)

Ключові спостереження з експериментів:

- **Запит `"animal"`** → тварини (cat, bear...) зверху, комахи (mosquito) і нерелевантні записи внизу. Embeddings коректно групують семантично близькі поняття.
- **Запит `"cat"`** → cat на першому місці, далі dog, bear, tiger — інші тварини.
- **Dot product vs cosine similarity** — дають дуже схожі результати (порядок може трохи відрізнятись, але загальна картина та сама).
- **Точний збіг тексту** → similarity ≈ 1 (модель детермінована для однакового input; регістр може давати незначну відмінність).
- **Експеримент з фактами про людину** (`data2.json`, речення про John) — запит `"How old is John?"` повертає `"John is 30 years old"` першим. Це демонструє **найпростіший прототип semantic search / RAG**: розбити текст на фрагменти → embed кожен → знайти найближчий фрагмент до запиту → це і є «відповідь».

## 5. Загальні висновки по розділу

1. **Embeddings** перетворюють будь-які дані (текст) у числові вектори, де геометрична близькість відповідає семантичній схожості.
2. OpenAI Embeddings API (`text-embedding-3-small`) — простий у використанні: один виклик `embeddings.create` для одного або багатьох рядків.
3. Без vector database можна самостійно зберігати й обробляти embeddings через звичайні JSON-файли — достатньо для навчальних і невеликих проєктів.
4. **Dot product** і **cosine similarity** — дві основні метрики similarity; на практиці дають подібні результати, cosine — незалежна від «довжини» вектора.
5. Semantic search на embeddings — це фактично **прототип RAG**: основа для майбутніх, більш складних тем курсу (чат-асистенти, повноцінний RAG).
6. Розділ завершується практичним закріпленням — самостійним проєктом рекомендаційної системи фільмів, що об'єднує всі вивчені техніки (генерація, збереження, порівняння embeddings).
