# OpenAI roles (ролі повідомлень)

У запитах до **Chat Completions API** важливим параметром є **`messages`** — масив об’єктів, кожен з яких має **роль** (`role`) і **вміст** (`content`).

Роль визначає, **хто «говорить»** у діалозі: користувач, системні інструкції чи відповідь моделі.

## Роль `user`

Використовується, коли **промпт надсилає користувач** системи — тобто ви самі задаєте питання або інструкцію.

Цю роль ми вже використовували раніше:

```typescript
messages: [
  { role: "user", content: "How many countries are in the world?" }
]
```

## Масив повідомлень

`messages` — це **масив**, тому в одному запиті можна передати **кілька повідомлень** з різними ролями. Це дозволяє задавати контекст і поведінку моделі перед основним запитом користувача.

## Роль `system`

**System role** налаштовує **те, як модель відповідає** — стиль, тон, формат, обмеження тощо.

### Простий приклад — зміна стилу

```typescript
messages: [
  {
    role: "system",
    content: "You respond like a cool bro."
  },
  {
    role: "user",
    content: "How many countries are in the world?"
  }
]
```

Навіть якщо **фактична відповідь** залишається схожою, **манера подачі** суттєво змінюється — наприклад, неформальний «bro»-стиль замість нейтрального тексту.

### Складніший приклад — формат відповіді (JSON)

System prompt можна використовувати для **структурованих відповідей**:

```typescript
messages: [
  {
    role: "system",
    content: `You respond like a cool bro.
You respond in JSON format with:
- "coolnessLevel": number from 1 to 10
- "answer": your answer`
  },
  {
    role: "user",
    content: "How many countries are in the world?"
  }
]
```

Модель може повернути, наприклад:

```json
{
  "coolnessLevel": 8,
  "answer": "There are about 195 countries in the world, bro!"
}
```

Таким чином **system role** — це спосіб «налаштувати» поведінку OpenAI: від легкого тону до суворих правил формату виводу.

**Аналогія з лекції:** уявіть ChatGPT перед дзеркалом, який каже собі: «Ти можеш це зробити. Ти — крута система» — system prompt задає саме таку «внутрішню установку» перед відповіддю.

## Роль `assistant`

**Assistant role** — для **відповідей, які повертає OpenAI**. Ми **не задаємо її вручну** в типовому запиті; вона приходить у відповіді API.

Якщо вивести повне повідомлення з відповіді:

```typescript
console.log(response.choices[0].message);
// { role: "assistant", content: "..." }
```

Поле `role` у відповіді буде **`assistant`** — це і є роль відповіді моделі.

Роль `assistant` також використовують у **багатокрокових діалогах**, коли ви передаєте історію чату назад у API (попередні відповіді моделі зберігаються з цією роллю).

## Три ролі — коротко

| Роль | Хто «говорить» | Призначення |
|------|----------------|-------------|
| **`user`** | Користувач | Промпт, питання, інструкція від людини |
| **`system`** | Система | Налаштування поведінки, стилю, формату відповіді |
| **`assistant`** | Модель | Відповіді OpenAI (у response або в історії чату) |

## Підсумок

1. Параметр **`messages`** — масив об’єктів `{ role, content }`.
2. **`user`** — повідомлення від користувача (ваш промпт).
3. **`system`** — інструкції для моделі: тон, стиль, формат (наприклад, JSON з конкретними полями).
4. **`assistant`** — відповіді моделі; у звичайному запиті приходить у `response.choices[0].message`.
5. System prompt сильно впливає на **як** відповідає модель, навіть якщо зміст залишається тим самим.
6. У **наступній лекції** — інші параметри запитів до OpenAI API.
