# Інші параметри OpenAI Chat API

У цій лекції розглядаються додаткові параметри **Chat Completions API**: `temperature`, `top_p`, `max_tokens`, `n`, `frequency_penalty`, `seed` та інші.

Їх можна налаштовувати двома способами:

1. **OpenAI Playground** — веб-інтерфейс на [platform.openai.com](https://platform.openai.com/playground), де видно ролі `system`, `user` і відповідь моделі (`assistant`).
2. **Код (API)** — частина параметрів доступна лише через програмний інтерфейс.

---

## Temperature (температура)

**Контролює випадковість** відповіді.

| Значення | Діапазон | Ефект |
|----------|----------|-------|
| `0` | 0–2 | Майже без випадковості — передбачувана відповідь |
| `2` | 0–2 | Максимальна випадковість — дуже різноманітна відповідь |

- **Вища температура** → більш різноманітні відповіді.
- **Вища температура** → вищий ризик **галюцинацій** (модель «видає» нісенітницю або вигадані факти).

**Приклад у Playground:** промпт «Say something cool in 200 words or less» з `temperature: 1` дає звичайну відповідь; з `temperature: 2` — набагато більш хаотичну.

---

## top_p

**Контролює різноманітність токенів** у відповіді (nucleus sampling).

| Значення | Діапазон | Ефект |
|----------|----------|-------|
| Низьке (напр. `0.2`) | 0–1 | Модель обирає лише найімовірніші токени — **більш поширені слова** |
| Високе | 0–1 | Більша різноманітність слів |

> **Увага:** OpenAI **не рекомендує** одночасно змінювати `temperature` і `top_p` в одному запиті.

Разом ці два параметри допомагають отримати **більш детерміновані** (передбачувані) результати — коли потрібно знати, чого очікувати від моделі.

---

## max_tokens

**Обмежує довжину відповіді** у токенах.

Якщо вказати, наприклад, `max_tokens: 50`, відповідь буде **обрізана** до вказаної кількості токенів — навіть якщо модель «хотіла» написати більше.

```typescript
const response = await openai.chat.completions.create({
  model: "gpt-4o-mini",
  messages: [{ role: "user", content: "Say something cool" }],
  max_tokens: 50,
});
```

---

## n (number of choices)

**Кількість варіантів відповіді** на один запит.

- За замовчуванням: `n: 1` — одна відповідь.
- `n: 2` — дві різні відповіді в масиві `choices`.

```typescript
console.log(response.choices[0].message.content); // перша відповідь
console.log(response.choices[1].message.content); // друга відповідь
```

---

## frequency_penalty

**Штраф за повторення токенів** — зменшує ймовірність повторного використання вже з’явившихся слів.

| Діапазон | Ефект |
|----------|-------|
| `-2` … `2` | Чим **вище** значення (напр. `1.5`), тим сильніше модель уникає повторів |

Корисно, коли потрібна більш різноманітна лексика без однакових фраз.

---

## seed

**«Підпис» запиту** для більш **детермінованих** відповідей.

Можна передати числове значення `seed` — модель намагатиметься давати стабільніші результати для однакових запитів.

> На момент лекції `seed` — **beta-функція**; з часом може стати повноцінною.

У відповіді також повертається **`system_fingerprint`** — ідентифікатор конфігурації системи. Якщо потрібно переконатися, що однакові запити дають однакові відповіді, можна орієнтуватися на цей fingerprint.

```typescript
const response = await openai.chat.completions.create({
  model: "gpt-4o-mini",
  messages: [{ role: "user", content: "Say something cool" }],
  seed: 42,
});
```

---

## Підсумок

| Параметр | Призначення |
|----------|-------------|
| `temperature` | Випадковість / креативність відповіді |
| `top_p` | Різноманітність обраних токенів |
| `max_tokens` | Максимальна довжина відповіді |
| `n` | Кількість варіантів відповіді |
| `frequency_penalty` | Уникнення повторення слів |
| `seed` | Детермінованість відповідей |

Параметрів багато, і за кілька хвилин їх повністю не освоїш. Найкращий спосіб зрозуміти їх — **експериментувати** в Playground та в коді й спостерігати, як змінюється поведінка моделі.
