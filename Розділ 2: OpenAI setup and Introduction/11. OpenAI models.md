# OpenAI models

Один із найважливіших параметрів при роботі з OpenAI API — і взагалі з AI — це **модель**.

## Що таке AI-модель?

Коротко: **модель — це спрощене представлення чогось великого**.

Аналогія з реального світу: є справжній автомобіль або будинок, а є їхні мініатюрні копії — це і є «моделі».

У контексті штучного інтелекту **модель** — це **набір алгоритмів**, які відображають реальну ситуацію. Ці алгоритми походять з **машинного навчання (machine learning)** і здатні:

- навчатися на великих обсягах даних;
- **генерувати** або **узагальнювати** дані, яких раніше не бачили;
- робити **прогнози** або **припущення** на основі наявних даних.

### Великі мовні моделі (LLM)

Особливо важливий випадок AI-моделей — **large language models (LLM)**, які зосереджені на **генерації тексту**.

У цьому курсі вже використовується одна з таких моделей — **GPT-3**.

## Моделі OpenAI

У [документації OpenAI](https://platform.openai.com/docs/models) (розділ Models) можна побачити повний список доступних моделей.

### Текстові моделі

| Модель | Опис |
|--------|------|
| **GPT-4** | Найпотужніша модель для генерації тексту та роботи з природною мовою |
| **GPT-4 Turbo** | Покращена версія GPT-4 |
| **GPT-3.5** | Найпопулярніша модель на момент запису курсу; має варіації (наприклад, `gpt-3.5-turbo`, `gpt-3.5-turbo-1106`) |

**Важливо:**

- GPT-4 і GPT-4 Turbo — **платні** моделі (без безкоштовного рівня), але навіть для навчання вони **досить дешеві**.
- Моделі швидко еволюціонують — на момент перегляду відео можуть бути доступні новіші версії.

### Context window (контекстне вікно)

Кожна модель має **context window** — обсяг тексту, який вона може обробити за один запит. Вимірюється в **токенах**.

Детальніше про токени — у **наступній лекції**.

### Інші моделі OpenAI

Крім текстових, OpenAI пропонує:

- **DALL-E** — генерація зображень за текстовим описом;
- **Text-to-Speech** — перетворення тексту в мовлення;
- та інші.

Рекомендується самостійно переглянути документацію OpenAI.

## Альтернативи OpenAI

Пізніше в курсі буде окремий розділ про **вихід за межі OpenAI** — використання моделей, які не є пропрієтарними рішеннями однієї компанії.

### Hugging Face

Одна з найкращих платформ для **open-source моделей** — [Hugging Face](https://huggingface.co/).

На сайті доступні моделі для різних задач:

- генерація тексту;
- обробка зображень;
- та багато інших.

Приклади популярних open-source моделей:

- **Llama** (Meta/Facebook) — потужна та популярна модель;
- інші моделі для text generation, які можна знайти в каталозі Hugging Face.

**Перевага:** можна створити акаунт, **завантажити модель локально** і використовувати її на власному комп’ютері — без залежності від API OpenAI.

## Підсумок

1. **Модель** — ключовий параметр при роботі з AI; це набір алгоритмів ML, що відображає реальну ситуацію.
2. **LLM** (наприклад, GPT-3, GPT-4) — моделі для генерації тексту.
3. OpenAI пропонує **GPT-4** (найпотужніша, платна), **GPT-3.5** (популярна) та інші моделі (DALL-E, TTS тощо).
4. Кожна модель має **context window** у токенах — скільки тексту вона обробляє за раз.
5. **Hugging Face** — джерело open-source моделей, які можна запускати локально.
6. Детальніше про **токени** — у наступній лекції.
