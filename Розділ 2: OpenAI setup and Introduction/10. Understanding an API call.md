# Розуміння API-виклику

Перед тим як рухатися далі, важливо з’ясувати, **що саме ми робимо** з локально встановленою бібліотекою OpenAI.

## Що ми насправді робили

У попередніх лекціях ми отримували **completions** (відповіді) від OpenAI. Але локально ми **не виконували жодних складних обчислень** — не генерували відповідь на своєму комп’ютері.

Ми просто використовували **клієнтську бібліотеку OpenAI** — обгортку для доступу до **OpenAI API**. Саме це і є суть того, що ми робили.

## Офіційні бібліотеки та HTTP

У [документації OpenAI](https://platform.openai.com/docs) (розділ API Reference) зазначено, що API можна викликати через дві офіційні бібліотеки:

- **Python**
- **Node.js**

Але це **не єдиний спосіб**. Будь-яка мова або інструмент, що вміє робити **HTTP-запити**, може звертатися до OpenAI API — адже по суті це звичайний HTTP-виклик.

### Альтернативні способи виклику

| Спосіб | Опис |
|--------|------|
| **Postman** | GUI для відправки HTTP-запитів |
| **curl** | Прямий HTTP-запит з терміналу |
| **request** (Node.js) | Будь-яка HTTP-бібліотека в Node.js |
| **OpenAI client** | Офіційна обгортка (зручніша, але не обов’язкова) |

**Висновок:** клієнтська бібліотека OpenAI — це зручність, а не вимога. Під капотом завжди йде звичайний HTTP POST-запит.

## API-ключ

Для ідентифікації користувача потрібен **OpenAI API key**. Без нього запит не пройде.

- Для простих прикладів коштів зазвичай не потрібно.
- У міру ускладнення запитів і зростання обсягу використання **потрібна буде оплата**.

## Структура відповіді

Навіть простий запит («скажи "This is a test"») повертає **не лише текст відповіді**. У відповіді є додаткові поля:

- **model** — яка модель обробила запит
- **prompt_tokens** — кількість токенів у запиті
- **completion_tokens** — кількість токенів у відповіді
- **total_tokens** — загальна кількість токенів

Тобто API-виклик складніший, ніж просто «питання → відповідь».

## Playground

Ті самі запити можна робити через **Playground** на сайті OpenAI — без коду, прямо в браузері.

У Playground справа відображаються **параметри запиту**:

- `model`
- `temperature`
- `top_p`
- та інші

Їх детальний розбір — у **наступній лекції**.

## Підсумок

1. Локальна бібліотека OpenAI — це **клієнт для HTTP API**, а не локальна AI-модель.
2. API можна викликати **будь-яким HTTP-клієнтом** (curl, Postman, request тощо).
3. Для автентифікації потрібен **API key**.
4. Відповідь містить **контент і метадані** (модель, токени).
5. Той самий запит можна відтворити в **Playground** на сайті OpenAI.
