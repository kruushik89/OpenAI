# Токени

Токени — одна з найважливіших тем у роботі з AI. Саме через них більшість AI-провайдерів (зокрема OpenAI) **рахують вартість** використання сервісу.

## Навіщо потрібні токени

1. **Оплата** — рахунок формується за кількістю токенів у запиті та відповіді.
2. **Обмеження моделі** — кількість токенів визначає, чи можна використати певну модель і скільки контексту вона обробить (context window).

### Приклад з попередньої лекції

Запит «скажи "This is a test"» повернув у метаданих `usage`:

- **13 prompt tokens** — токени в запиті;
- **5 completion tokens** — токени у відповіді.

Тобто навіть короткий обмін текстом уже має конкретну «вартість» у токенах.

## Що таке токен?

**Токен** — це фрагмент тексту, на який модель розбиває вхідні та вихідні дані. Кожному токену відповідає числовий **ID**.

### OpenAI Tokenizer

Найзручніший спосіб побачити токенізацію — інструмент [OpenAI Tokenizer](https://platform.openai.com/tokenizer): вводите текст і одразу бачите, як він розбитий на токени (з кольоровим виділенням).

**Приклад:** `How are you today?`

- Слова англійською часто є окремими токенами (наприклад, `today`).
- Якщо додати пробіл і ще слово — `today` залишається токеном, а наступне слово — іншим.
- Один `?` — один токен; два `??` — один спільний токен; три `???` — знову окремий токен.

**Загальне правило (не абсолютне):** один токен ≈ **4 символи**, але довжина сильно залежить від мови, пунктуації та конкретного слова.

## Алгоритми токенізації

Щоб глибше зрозуміти, як утворюються токени, варто почитати матеріали на [Hugging Face](https://huggingface.co/). Два ключові алгоритми:

| Алгоритм | Назва |
|----------|-------|
| **BPE** | Byte Pair Encoding |
| **WordPiece** | WordPiece |

## Токени та ціноутворення OpenAI

На сторінці [openai.com/pricing](https://openai.com/pricing) вказано ціну **за 1000 токенів (1K tokens)** для кожної моделі.

На момент запису курсу:

- **GPT-4 Turbo** — найпотужніша і найдорожча модель.
- **GPT-3.5** — значно дешевша (лише частка долара за 1000 токенів); ціна нещодавно знизилась.

### Відстеження витрат

У акаунті OpenAI (розділ **Usage**) можна побачити фактичне споживання та очікувану оплату.

Для **нових акаунтів** OpenAI надає **грант $5 на 3 місяці** — зручно для тестування API без негайної оплати.

## Підрахунок токенів у коді

Для коротких промптів це не критично, але далі в курсі будуть **великі промпти**, і потрібно буде точно знати їхній розмір у токенах.

Для цього використовують бібліотеку **tiktoken**:

```bash
npm install tiktoken
```

```typescript
import { encoding_for_model } from "tiktoken";

function encodePrompt() {
  const prompt = "How are you today?";
  const encoder = encoding_for_model("gpt-3.5-turbo");
  const tokens = encoder.encode(prompt);

  console.log(tokens);
  // Масив числових ID токенів, напр. 5 токенів для цього промпту
}
```

Результат — масив **token IDs**, такий самий, як на сайті OpenAI Tokenizer. Функція `encoding_for_model` приймає назву моделі (наприклад, `gpt-3.5-turbo`), щоб токенізація відповідала тій, яку використовує API.

## Підсумок

1. **Токени** — одиниці, в яких AI-моделі обробляють текст; за ними формується оплата.
2. Перевірити токенізацію можна через **OpenAI Tokenizer**; у відповіді API — поля `prompt_tokens`, `completion_tokens`, `total_tokens`.
3. Ціна на [openai.com/pricing](https://openai.com/pricing) вказана **за 1K токенів**; GPT-3.5 дешевший за GPT-4 Turbo.
4. Нові акаунти отримують **$5 гранту** на 3 місяці.
5. У коді токени рахують через **tiktoken** (`encoding_for_model` + `encode`).
6. У **наступній лекції** — інші параметри запитів до OpenAI API.
