# Whisper: транскрипція, переклад і Text-to-Speech

У цьому уроці — **практична реалізація** трьох аудіо-функцій OpenAI API: транскрибування, переклад аудіо в англійську та синтез мовлення (TTS).

---

## Підготовка проєкту

Створюється новий файл, наприклад `audio-tts.ts`.

### Імпорти

```ts
import OpenAI from "openai";
import { writeFileSync, createReadStream } from "fs";
```

- **`createReadStream`** — для передачі аудіофайлу в API (транскрипція та переклад).
- **`writeFileSync`** — для збереження згенерованого MP3 на диск (TTS).

Далі — екземпляр OpenAI client і три функції. Скрипт підключається в `package.json` і запускається через `npm start`.

### Тестові файли

Для прикладів використовуються готові аудіозаписи (доступні як ресурси курсу):

- **`audio-sample.mp4`** — англійське мовлення для транскрипції;
- **`french-sample`** — французьке мовлення для перекладу.

Можна підставити будь-який свій файл з мовленням.

---

## API для аудіо

Аудіо — **окремий розділ** OpenAI API (`openai.audio`), не `chat.completions`. У цьому уроці три endpoint'и:

| Функція | Endpoint | Модель |
|---------|----------|--------|
| Транскрипція | `audio.transcriptions.create` | `whisper-1` |
| Переклад | `audio.translations.create` | `whisper-1` |
| Озвучення | `audio.speech.create` | `tts-1` |

На момент уроку для Whisper доступна лише **`whisper-1`**; пізніше можуть з’явитися інші моделі.

---

## Функція 1: `createTranscription` (Speech-to-Text)

**Транскрипція** — подаємо аудіо, отримуємо текст (модель «слухає» запис і повертає розпізнаний текст).

### Параметри запиту

```ts
const response = await openai.audio.transcriptions.create({
  file: createReadStream("./audio-sample.mp4"),
  model: "whisper-1",
  // language: "en" — опційно; модель часто визначає мову сама
});
```

| Параметр | Опис |
|----------|------|
| `file` | Потік з аудіофайлу через `createReadStream` |
| `model` | `"whisper-1"` |
| `language` | Опційно — підказка мови (наприклад `"en"`) |

### Результат

У відповіді — поле **`text`** з розпізнаним текстом з аудіо. Виводимо через `console.log(response)`.

---

## Функція 2: `translate` (аудіо → англійський текст)

**Переклад** використовує той самий Whisper, але інший endpoint — **`audio.translations`**.

### Важливо

Переклад завжди **в англійську**. Вхідне аудіо має бути **не англійською** (у прикладі — французька запис з Wikipedia).

```ts
const response = await openai.audio.translations.create({
  file: createReadStream("./french-sample.mp4"), // шлях до французького зразка
  model: "whisper-1",
});
```

Можна записати себе рідною мовою (якщо це не англійська) і перевірити результат.

### Результат

API повертає **англійський текст** — те, що було сказано в оригінальному аудіо, перекладене англійською.

## Функція 3: Text-to-Speech (TTS)

**Text-to-Speech** — подаємо текст, отримуємо **аудіофайл** з озвученням.

### Приклад тексту

Короткий опис Франції (для демонстрації):

> France is a country from Europe. It is located in Western Europe and is known for its rich history, world famous cuisine and stunning architecture.

### Параметри запиту

```ts
const response = await openai.audio.speech.create({
  input: sampleText,
  voice: "alloy",
  model: "tts-1",
  response_format: "mp3",
});
```

| Параметр | Приклад | Опис |
|----------|---------|------|
| `input` | текст для озвучення | Зміст, який «прочитає» модель |
| `voice` | `"alloy"` | Голос; у документації TTS є **зразки для кожного голосу** — можна обрати найзручніший |
| `model` | `"tts-1"` | Стандартна модель; **`tts-1-hd`** — платна HD-версія |
| `response_format` | `"mp3"` | Формат вихідного файлу |

### Збереження відповіді на диск

На відміну від зображень, TTS **не повертає URL** — це **сира бінарна відповідь**. Її треба перетворити в буфер і записати у файл:

```ts
const buffer = Buffer.from(await response.arrayBuffer());
writeFileSync("france.mp3", buffer);
```

Після `npm start` з’являється файл **`france.mp3`**. Прослуховування підтверджує, що озвучення відповідає вхідному тексту — якість TTS вражає.

## Підсумок уроку

| Можливість | Вхід | Вихід | Модель |
|------------|------|-------|--------|
| Транскрипція | Аудіофайл | Текст (та ж мова) | `whisper-1` |
| Переклад | Аудіо (не англійською) | Текст англійською | `whisper-1` |
| TTS | Текст | Аудіофайл (MP3) | `tts-1` |

Усі три сценарії добре підходять для застосунків: субтитри, переклад подкастів, озвучка контенту, голосові асистенти тощо.
