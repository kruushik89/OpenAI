# Генерація зображень: практика з DALL-E 2 і DALL-E 3

У цьому уроці — **практична реалізація** генерації зображень через OpenAI API: від простого запиту з DALL-E 2 до збереження файлу локально та використання DALL-E 3.

---

## Підготовка проєкту

У проєкті **Basics** створюється структура для роботи з іншими сервісами OpenAI:

```
Basics/
└── others/
    └── images.ts
```

У файлі `images.ts`:

1. Імпортуємо клієнт OpenAI (як і для текстових моделей).
2. Створюємо екземпляр `OpenAI` client.
3. Додаємо функції генерації зображень.
4. Підключаємо скрипт у `package.json` і запускаємо через `npm start`.

### Повний код `images.ts`

```ts
import { writeFileSync } from "fs";
import OpenAI from "openai";

const openai = new OpenAI();

async function generateFreeImage() {
  const response = await openai.images.generate({
    prompt: "A beautiful sunset over a calm ocean",
    model: "gpt-image-1",
    n: 1,
    size: "1024x1024",
  });

  const rawImage = response.data?.[0]?.b64_json;

  if (rawImage) {
    const filename = "sunset.png";
    writeFileSync(filename, Buffer.from(rawImage, "base64"));
    console.log(`Image saved to ${filename}`);
  }
}

generateFreeImage();
```

Скрипт у `package.json`:

```json
"start": "node -r ts-node/register --env-file=.env src/others/images.ts"
```

Після `npm start` у корені проєкту з’являється файл `sunset.png`.

## API для зображень

Генерація зображень — **окремий endpoint**, не `chat.completions`. Виклик:

```ts
const response = await openai.images.generate({ ... });
```

У наступному уроці розглянуться також **редагування** зображень і створення **варіацій** (variations).

---

## Функція 1: `generateFreeImage` (DALL-E 2, URL)

Перша функція використовує **DALL-E 2** — її можна запускати на **безкоштовних кредитах** API без поповнення балансу.

### Параметри запиту

| Параметр | Приклад | Опис |
|----------|---------|------|
| `prompt` | `"a photo of a cat on a mat"` | Текстовий опис бажаного зображення |
| `model` | `"dall-e-2"` | Модель генерації |
| `size` | найменший доступний розмір | Роздільність зображення |
| `n` | `1` | Кількість згенерованих зображень у відповіді |

Для **DALL-E 3** (у наступних прикладах) додатково доступні:

- `style` — `"natural"` або `"vivid"` (більш гіперреалістичний вигляд);
- `quality` — `"standard"` або `"HD"`.

### Відповідь API

Після `console.log(response)` отримуємо об’єкт із полем `data` — **масив**. Якщо `n: 1`, у масиві один елемент з полем **`url`**.

Зображення можна відкрити, перейшовши за цим URL у браузері. DALL-E 2 дає простішу якість — для навчання цього достатньо.

## Функція 2: `generateFreeLocalImage` (збереження на диск)

Друга функція — та сама логіка, але зображення **зберігається локально**, а не лише за URL.

### Ключова зміна: `response_format`

```ts
response_format: "b64_json"
```

Замість URL API повертає **base64-рядок** у полі `b64_json` елемента масиву `data`.

### Збереження файлу (Node.js)

```ts
import { writeFileSync } from "fs";

const rawImage = response.data?.[0]?.b64_json;

if (rawImage) {
  writeFileSync("cat-mat.png", Buffer.from(rawImage, "base64"));
}
```

- Перевіряємо, що `b64_json` визначений (тип може бути `string | undefined`).
- `writeFileSync` записує буфер, отриманий з base64, у файл (наприклад, `cat-mat.png`).
- Після `npm start` файл з’являється в проєкті — зображення **на диску**, а не тільки в інтернеті.

## Функція 3: `generateAdvancedImage` (DALL-E 3)

Третя функція демонструє **DALL-E 3** — потрібен **платний баланс OpenAI API** (не підписка ChatGPT Plus, а саме **API billing** з поповненим кредитом).

### Параметри

| Параметр | Значення | Примітка |
|----------|----------|----------|
| `prompt` | `"photo of a city at night with skyscrapers"` | Детальніший опис |
| `model` | `"dall-e-3"` | Краща якість |
| `quality` | `"HD"` | Висока деталізація |
| `size` | більший розмір | Ніж у DALL-E 2 |
| `response_format` | `"b64_json"` | Збереження локально |

Файл зберігається аналогічно (наприклад, `city.png`). Результат DALL-E 3 **помітно кращий** за DALL-E 2 — більш реалістичне нічне місто з хмарочосами.

## Порівняння підходів

| Аспект | DALL-E 2 | DALL-E 3 |
|--------|----------|----------|
| Білінг | Безкоштовні кредити API | Потрібне поповнення балансу |
| Якість | Нижча, «простіше» зображення | HD, значно краща деталізація |
| Формат відповіді | `url` (за замовчуванням) або `b64_json` | Те саме |
| Збереження | URL у браузері або файл через base64 | Файл через base64 |

---

## Підсумок уроку

1. Генерація зображень — через **`openai.images.generate`**, не через чат.
2. **DALL-E 2** — безкоштовний старт: prompt, model, size, `n`.
3. Відповідь містить **`data[]`** з **`url`** або **`b64_json`**.
4. Локальне збереження — `response_format: "b64_json"` + `writeFileSync` + `Buffer.from(..., "base64")`.
5. **DALL-E 3** + **HD** — краща якість, але потрібен платний API-акаунт.
