# Редагування зображень: варіації та inpainting

У цьому уроці продовжуємо роботу в `others/images.ts` і розглядаємо **два додаткові методи** Images API: створення **варіацій** існуючого зображення та **редагування** (inpainting) за допомогою маски.

---

## Важливі зауваження

- Згенеровані файли (`city.png`, маски, результати) мають лежати **на верхньому рівні проєкту** — поруч із `package.json`, щоб `createReadStream` їх коректно читав.
- Для варіацій і редагування зображення має бути у форматі **PNG**.
- **Генерація зображень значно дорожча**, ніж робота з текстовими моделями — варто обмежувати кількість експериментів.
- Варіації та редагування доступні з **DALL-E 2** — можна працювати **без платного балансу** API (на відміну від DALL-E 3 для генерації).

---

## 1. Варіації зображення — `createVariation`

Якщо вже є згенероване зображення (наприклад, `city.png` з попереднього уроку), можна отримати **нову варіацію** на його основі — схоже зображення з іншими деталями.

### Функція `generateImageVariation`

```ts
import { createReadStream } from "fs";

const response = await openai.images.createVariation({
  image: createReadStream("city.png"),
  model: "dall-e-2",
  response_format: "b64_json",
  n: 1,
});
```

### Ключові параметри

| Параметр | Значення | Опис |
|----------|----------|------|
| `image` | `createReadStream("city.png")` | Вхідне зображення; у Node.js передається через потік читання |
| `model` | `"dall-e-2"` | Модель для варіацій |
| `response_format` | `"b64_json"` | Збереження локально (не URL) |
| `n` | `1` | Кількість варіацій у відповіді |

### Збереження результату

Як і раніше — витягуємо `b64_json` з `response.data[0]` і записуємо через `writeFileSync`:

```ts
writeFileSync("city-variation.png", Buffer.from(rawImage, "base64"));
```

Після `npm start` з’являється файл `city-variation.png` — варіація вихідного зображення. Якість може виглядати гірше за оригінал (обмеження DALL-E 2), але механізм працює коректно.

---

## 2. Редагування зображення — `edit` (inpainting)

Складніший сценарій: є зображення міста (`city.png`), і потрібно **додати щось у небо** — наприклад, грозу.

### Функція редагування

```ts
const response = await openai.images.edit({
  image: createReadStream("city.png"),
  mask: createReadStream("city-mask.png"),
  prompt: "add a thunderstorm to the city",
  model: "dall-e-2",
  response_format: "b64_json",
});
```

### Обов’язкові параметри

| Параметр | Опис |
|----------|------|
| `image` | Вихідне зображення (PNG) |
| `prompt` | Текстовий опис того, що додати або змінити |
| `mask` | Маска з **прозорою** областю, яку має відредагувати API |

> Навіть якщо TypeScript позначає `mask` як опційний, **на практиці він потрібен** для коректного редагування.

### Що таке маска

Маска — окремий PNG-файл того ж розміру, що й оригінал. Область, яку треба змінити (наприклад, небо), має бути **прозорою**; решта — непрозора. OpenAI «малює» лише в прозорих зонах.

---

## 3. Створення маски в GIMP

Безкоштовний редактор **GIMP** підходить для підготовки маски:

1. Відкрити `city.png` у GIMP.
2. **Layer → Transparency → Add Alpha Channel** — додати альфа-канал.
3. Інструмент **Free Select (Lasso)** — виділити область для редагування (наприклад, небо).
4. **Edit → Clear** (або Delete) — видалити виділену область, зробивши її прозорою.
5. **File → Export As** → зберегти як `city-mask.png` (саме **Export**, не Save).

Після цього `city-mask.png` використовується в коді через `createReadStream`.

---

## 4. Результат редагування

Збереження аналогічне попереднім прикладам — файл `city-edited.png`:

```ts
writeFileSync("city-edited.png", Buffer.from(rawImage, "base64"));
```

Після запуску `npm start` API додає грозу в небо на зображенні міста. Результат не ідеальний, але демонструє робочий **inpainting**-пайплайн.

---

## Порівняння методів Images API

| Метод | API-виклик | Вхід | Призначення |
|-------|------------|------|-------------|
| Генерація | `images.generate` | `prompt` | Нове зображення з тексту |
| Варіація | `images.createVariation` | `image` (PNG) | Схожа варіація існуючого файлу |
| Редагування | `images.edit` | `image` + `mask` + `prompt` | Зміна лише в прозорій зоні маски |

---

## Підсумок уроку

1. **Варіації** — `createVariation` + `createReadStream` для PNG-файлу; `n` задає кількість варіантів.
2. **Редагування** — `edit` + вихідне зображення + **маска з прозорою областю** + prompt.
3. Маску можна зробити в GIMP: alpha channel → виділення → clear → export as PNG.
4. DALL-E 2 достатньо для варіацій і редагування без платного API-балансу.
5. Файли зображень і масок — **поруч із `package.json`**.
