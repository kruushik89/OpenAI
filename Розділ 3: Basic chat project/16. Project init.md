# Ініціалізація проєкту чату

У цьому уроці ми робимо **базове налаштування** нового чат-застосунку — окремого від попереднього проєкту `basics`.

## Структура проєкту

1. Створюємо нову папку `chat-app` і ініціалізуємо npm-проєкт (`npm init -y`), щоб з’явився `package.json`.
2. Створюємо папку `src` — у ній буде весь вихідний код.

## Залежності

### Dev-залежності (TypeScript)

```bash
npm install -D typescript ts-node @types/node
```

- **typescript** — компілятор TypeScript;
- **ts-node** — запуск `.ts` файлів без окремої компіляції;
- **@types/node** — типи для Node.js (зокрема для `process`).

### Основні залежності

```bash
npm install openai tiktoken
```

- **openai** — офіційний SDK для OpenAI API;
- **tiktoken** — підрахунок довжини промптів у токенах (як у попередньому розділі).

## Конфігураційні файли

### `.env`

Файл змінних оточення для зберігання **OpenAI API key**. Можна скопіювати з іншого проєкту курсу.

### `.gitignore`

Щоб Git не відстежував зайве:

- `.env` — секрети не потрапляють у репозиторій;
- `node_modules` — залежності не комітимо.

### `tsconfig.json`

Без нього VS Code не «бачить» типи Node (наприклад, для `process`), навіть якщо `@types/node` встановлено. У курсі використовується конфіг, схожий на попередній проєкт:

| Опція | Значення | Призначення |
|--------|----------|-------------|
| `target` | `ES2022` | Цільова версія JavaScript |
| `module` | `commonjs` | Система модулів Node |
| `rootDir` | `src` | Корінь вихідного коду |
| `outDir` | `dist` | Вихід після компіляції (якщо потрібно) |
| `strict` | `true` | Суворий режим TypeScript |
| `esModuleInterop` | `true` | Сумісність з CommonJS-модулями |
| `useUnknownInCatchVariables` | `false` | У `catch` змінна має тип `Error`, а не `unknown` |

Код можна **виконувати напряму через ts-node**, без окремого кроку збірки в `.js`.

## Перший файл коду

У `src/basic-chat/index.ts` створюємо точку входу. Назва папки **`basic-chat`** — щоб відокремити простий чат від майбутнього, складнішого **`functions-chat`**.

На цьому етапі достатньо перевірити завантаження ключа:

```typescript
console.log(process.env.OPENAI_API_KEY);
```

## npm-скрипт `start`

У `package.json` додаємо скрипт запуску. **ts-node сам не підвантажує `.env`**, тому спочатку запускаємо **Node**, а вже через нього — ts-node:

```json
"start": "node -r ts-node/register --env-file=.env src/basic-chat/index.ts"
```

- `-r ts-node/register` — реєстрація ts-node для виконання TypeScript;
- `--env-file=.env` — завантаження змінних оточення з файлу `.env`;
- далі — шлях до `index.ts`.

Запуск:

```bash
npm start
```

Якщо в консолі з’являється API key — **налаштування завершено**, можна переходити до реалізації чату в наступному уроці.
