# Базова реалізація чату

У цьому уроці починаємо **будувати простий чат-застосунок** у `src/basic-chat/index.ts` — без зовнішніх бібліотек для вводу, лише вбудовані можливості Node.js та SDK OpenAI.

## Підключення OpenAI

1. Імпортуємо клієнт:

```typescript
import OpenAI from "openai";
```

2. Створюємо екземпляр:

```typescript
const openai = new OpenAI();
```

Ключ API підтягується з `.env` (налаштовано на попередньому уроці).

## Ввід користувача через `process.stdin`

Потрібен **безперервний діалог** у терміналі. Для цього слухаємо стандартний ввід через об’єкт `process` з Node.js — **без додаткових бібліотек** (наприклад, `readline`).

```typescript
process.stdin.addListener("data", async (data) => {
  // обробка вводу
});
```

На подію `data` викликається **async-функція**, яка отримує введені дані.

## Обробка запиту

1. Перетворюємо ввід на рядок і обрізаємо пробіли:

```typescript
const message = data.toString().trim();
```

2. Викликаємо Chat Completions API:

```typescript
const response = await openai.chat.completions.create({
  model: "gpt-4o-mini",
  messages: [
    { role: "system", content: "You are a helpful assistant." },
    { role: "user", content: message },
  ],
});
```

- **system** — задає поведінку асистента;
- **user** — поточне повідомлення користувача.

3. Виводимо відповідь у консоль:

```typescript
console.log("response", response.choices[0].message.content);
```

## Повний код `src/basic-chat/index.ts`

```typescript
import OpenAI from "openai";

const openai = new OpenAI();

process.stdin.addListener("data", async (data) => {
    const message = data.toString().trim();
    const response = await openai.chat.completions.create({
        model: "gpt-4o-mini",
        messages: [
            { role: "system", content: "You are a helpful assistant." },
            { role: "user", content: message }
        ],
    });
    console.log("response", response.choices[0].message.content);
});
```

## Перевірка

Запуск:

```bash
npm start
```

Приклади запитів:

- «How are you today?» — асистент відповідає.
- «Who was Albert Einstein?» — отримуємо коректну відповідь.

**Перша версія працює** для окремих питань.

## Обмеження поточної реалізації

**Немає контексту діалогу (історії повідомлень).** Кожен запит надсилається як новий, із лише одним user-повідомленням.

Якщо спочатку запитати про Альберта Ейнштейна, а потім написати «Tell me more about him», модель **не зрозуміє**, про кого йдеться — вона не бачить попередніх повідомлень.

## Що далі

У **наступному уроці** додамо **збереження історії чату**, щоб асистент міг вести послідовний діалог з урахуванням контексту.
