# Конфігурація контексту (історії повідомлень)

У попередньому уроці ми зібрали **базовий чат**, який працює для окремих запитів, але **не зберігає контекст діалогу**. Кожен запит надсилається як новий — модель не бачить попередніх повідомлень.

У цьому уроці додаємо **історію повідомлень**, щоб асистент міг вести послідовний діалог.

## Ключова ідея

У запиті `chat.completions.create` поле **`messages`** — це масив повідомлень, що складають **всю розмову на даний момент**. На кожному новому запиті потрібно:

1. Додавати повідомлення користувача до історії.
2. Надсилати всю історію в API.
3. Додавати відповідь асистента до історії.

## Масив `context`

Виносимо історію на рівень вище — окремий масив `context`:

```typescript
const context: OpenAI.Chat.Completions.ChatCompletionMessageParam[] = [
  { role: "system", content: "You are a helpful assistant." },
];
```

- **system**-промпт завжди залишається першим елементом.
- Типізація в TypeScript необов'язкова, але бажана — `ChatCompletionMessageParam[]`.

## Функція `createChatCompletion`

Логіку запиту до API виносимо в окрему async-функцію:

```typescript
async function createChatCompletion() {
  const response = await openai.chat.completions.create({
    model: "gpt-4o-mini",
    messages: context,
  });

  const responseMessage = response.choices[0].message;
  context.push(responseMessage);
  console.log(`${responseMessage.role}: ${responseMessage.content}`);
}
```

- У `messages` передаємо весь масив `context`.
- Після отримання відповіді **додаємо її в контекст** — цілий об'єкт повідомлення з роллю `assistant`.

## Обробка вводу користувача

У обробнику `process.stdin` перед викликом API додаємо user-повідомлення:

```typescript
process.stdin.addListener("data", async (data) => {
  const userInput = data.toString().trim();
  context.push({ role: "user", content: userInput });
  await createChatCompletion();
});
```

Порядок: **user → запит → assistant** — і так на кожній ітерації.

## Вивід відповіді

Виводимо **роль і `content`** відповіді, а не весь об'єкт:

```typescript
console.log(`${responseMessage.role}: ${responseMessage.content}`);
```

## Повний код `src/basic-chat/index.ts`

```typescript
import OpenAI from "openai";

const openai = new OpenAI();

const context: OpenAI.Chat.Completions.ChatCompletionMessageParam[] = [
  { role: "system", content: "You are a helpful assistant." },
];

async function createChatCompletion() {
  const response = await openai.chat.completions.create({
    model: "gpt-4o-mini",
    messages: context,
  });

  const responseMessage = response.choices[0].message;
  context.push(responseMessage);
  console.log(`${responseMessage.role}: ${responseMessage.content}`);
}

process.stdin.addListener("data", async (data) => {
  const userInput = data.toString().trim();
  context.push({ role: "user", content: userInput });
  await createChatCompletion();
});
```

## Перевірка

Запуск:

```bash
npm start
```

Приклади, що підтверджують роботу контексту:

1. Запитати про особистість (наприклад, Альберта Ейнштейна), потім написати **«Tell me more about him»** — модель розуміє, про кого йдеться.
2. Ланцюжок обчислень: «one plus one is two» → «and three more, it's five» → «times two» — модель **зберігає весь контекст** діалогу.

## Проблема зростання контексту

Діалог працює, але є **недолік**: при кожній новій відповіді ми **перенадсилаємо всю історію** назад до OpenAI. Чим довша розмова, тим:

- більший розмір запиту;
- більше токенів;
- вища вартість API-викликів.

Навіть якщо нам потрібні лише останні 2–3 (або 10) повідомлень, зараз відправляється все.

## Що далі

У **наступному уроці** розглянемо, як **видаляти повідомлення з контексту** після перевищення певного порогу токенів, щоб контролювати розмір і вартість запитів.
